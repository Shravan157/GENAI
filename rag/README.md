# RAG Basics

## What is this?
This folder covers the first steps of RAG: loading documents and splitting text into chunks. It does not build a full vector database pipeline, but it prepares the data needed for one. In short: RAG starts by turning external content into clean chunks.

## Why does it exist?
- Use when you want model answers based on files, PDFs, CSVs, or webpages.
- Use this README when you open this folder and want a quick overview before reading code.
